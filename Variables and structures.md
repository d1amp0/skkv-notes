## Область видимости и инициализация

### Инициализация

 - Инициализацией называется создание объекта вместе с присваиванием ему значения.
 - Если просто сделать `int a;`, то объект не будет инициализирован, но под него будет выделена память.

### Глобальные объекты

- Объявляются вне функций и видны ВЕЗДЕ.
- Обычно размещаются в секции данных или BSS.
- Если явной инициализации нет, инициализируются нулем.
- Не рекомендуется использовать глобальные переменные, за исключением констант.

### Локальные объекты

- Объявляются внутри блоков и функций и видны ВНУТРИ БЛОКА.
- Обычно живут на стеке, хотя стандарт формулирует это через время хранения, а не через конкретный механизм.
- Без явной инициализации содержат МУСОР.

```c
int g; // Глобальная переменная

int main(void) {
    int a;     // Локальная переменная, не инициализирована
    int b = 0; // Локальная переменная, инициализирована
    return 0;
}
```

## Структуры (struct)

`struct` позволяет объединять несколько полей в один составной тип.

Пример объявления:

```c
struct S {
    int x;
    char c;
};
```

Что важно помнить:
- внутри `struct` в C можно объявлять только поля;
- после определения структуры ставится `;`;
- вместе с определением типа можно сразу объявить объекты.

```c
struct S {
    int x;
    char c;
};

// Создание объекта:
struct S strct;
```

Если хочется использовать тип без постоянного написания слова `struct`, можно применять ключевое слово `typedef`:

```c
typedef struct S {
    int x;
    char c;
} S;

// Теперь можно писать:
S c;
```

## Выравнивание и компоновка структур (NOT EDITED)

Поля структуры могут размещаться не вплотную, а с учетом требований выравнивания.

- У каждого типа может быть требование по кратности адреса.
- Выравнивание структуры обычно определяется самым "требовательным" полем.
- Между полями и в конце структуры компилятор может вставлять **padding**.

Это влияет на:

- размер структуры;
- совместимость бинарных форматов;
- взаимодействие с оборудованием и сетью;
- производительность на некоторых архитектурах.

Поэтому порядок полей — не косметика, а часть проектирования данных.

## Ключевые слова и модификаторы

### `static`

У `static` несколько ролей, и их важно не путать.

- Для глобального объекта или функции `static` ограничивает видимость текущим файлом.
- Для локальной переменной `static` меняет время жизни: объект существует всю программу, но виден только внутри блока.

```c
static int counter;

int f(void) {
    static int calls = 0;
    return ++calls;
}
```

### `enum`

- `enum` задает перечислимый тип с набором именованных констант.
- Полезен, когда значения образуют конечный набор состояний, режимов или кодов.

```c
// Два примера для enum с заданием значений и без
enum Color {
    COLOR_RED,
    COLOR_GREEN,
    COLOR_BLUE
};

enum Color {
	COLOR_RED = 1,
	COLOR_GREEN = 2,
	COLOR_BLUE = 3
}
```

### `const`

- Объект, объявленный с `const`, нельзя изменять через это имя.
- `const` не делает значение "магически помещенным в read-only память" во всех случаях, но фиксирует контракт на запись.
- Особенно важен в интерфейсах функций и при работе с указателями.

### `restrict`

- Ключевое слово для указателей, важное прежде всего для оптимизации.
- Сообщает компилятору, что в рамках данного участка доступа этот указатель — единственный способ обратиться к соответствующему объекту.
- Нарушение обещания `restrict` ведет к undefined behaviour.
- Часто встречается в реализациях функций из стандартных библиотек

### `constexpr`

- Это уже не C, а C++.
- Она является константой времени компиляции и позволяет использовать ее там, где нужен compile-time constant.

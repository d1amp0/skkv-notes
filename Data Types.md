--------------------------------------------------------------------------------
В языке C тип определяет, как интерпретируется память, какие операции допустимы над значением и какие соглашения ввода-вывода нужно соблюдать.

## Базовые типы

### `int`

- Базовый знаковый целочисленный тип.
- Эквивалентные записи: `int` = `signed` = `signed int`.
- Минимальный гарантированный диапазон: `-32767 ... 32767`.
- Это означает, что тип `int` должен иметь не менее 16 бит значения.

```c
int a = 239;
```

### `void`

- Обозначает отсутствие значения.
- Используется как тип возвращаемого значения функции без результата.
- В списке параметров `void` означает, что функция не принимает аргументов.

```c
void log_message(void) {
    printf("ok\n");
}
```

## Дополнительные целочисленные типы

### `unsigned int`

- Беззнаковая версия `int`.
- Эквивалентные записи: `unsigned` = `unsigned int`.
- Размер совпадает с `int`: `sizeof(unsigned int) == sizeof(int)`.
- Минимальный гарантированный диапазон: `0 ... 65535`.
- Переполнение для беззнаковых типов определено стандартом как арифметика по модулю.

### `short`

- Эквивалентные записи: `short` = `short int` = `signed short` = `signed short int`.
- Беззнаковая форма: `unsigned short` = `unsigned short int`.
- По стандарту `sizeof(short) <= sizeof(int)`.
- Минимальный гарантированный диапазон для знаковой формы: `-32767 ... 32767`.
- Минимальный гарантированный диапазон для беззнаковой формы: `0 ... 65535`.
- Это тоже как минимум 16-битный целый тип.

### `long`

- Эквивалентные записи: `long` = `long int` = `signed long` = `signed long int`.
- Беззнаковая форма: `unsigned long` = `unsigned long int`.
- По стандарту `sizeof(long) >= sizeof(int)`.
- Минимальный гарантированный диапазон для знаковой формы: `-2147483647 ... 2147483647`.
- Минимальный гарантированный диапазон для беззнаковой формы: `0 ... 4294967295`.
- Это как минимум 32-битный целый тип.
- Точный размер зависит от реализации, поэтому `long` не используют как тип "строго известной ширины".

### `long long`

- Эквивалентные записи: `long long` = `long long int` = `signed long long` = `signed long long int`.
- Беззнаковая форма: `unsigned long long` = `unsigned long long int`.
- По стандарту `sizeof(long long) >= sizeof(long)`.
- Минимальный гарантированный диапазон для знаковой формы: `-9223372036854775807 ... 9223372036854775807`.
- Минимальный гарантированный диапазон для беззнаковой формы: `0 ... 18446744073709551615`.
- Это как минимум 64-битный целый тип.

### `char`

- `char` имеет размер ровно `1` байт: `sizeof(char) == 1`.
- При этом байт в стандарте C не обязан быть равен 8 битам, но обязан содержать как минимум 8 бит: `CHAR_BIT >= 8`.
- `char`, `signed char` и `unsigned char` — это три разных типа.
- У обычного `char` знаковость не фиксирована стандартом: он может вести себя либо как `signed char`, либо как `unsigned char`.
- Для `signed char` минимальный гарантированный диапазон: `-127 ... 127`.
- Для `unsigned char` минимальный гарантированный диапазон: `0 ... 255`.
- Если нужен именно числовой смысл, лучше явно использовать `signed char` или `unsigned char`.

### `_Bool и bool`

- `_Bool` появился в C99.
- `bool` в C — это макрос из `<stdbool.h>`, который разворачивается в `_Bool`.
- Хранит логическое значение: по смыслу `0` или `1`.
- Минимально `_Bool` должен уметь хранить значения `0` и `1`; точный `sizeof(_Bool)` стандартом не фиксирован.
- Имена `bool`, `true`, `false` традиционно дает `<stdbool.h>`.

### Типы фиксированного размера

- Как можно было понять выше, типы `int`, `short`, `long`, `char` не гарантируют определённого значения, а лишь диапазон. Для этого могут помочь типы фиксированного размера.
- `<stdint.h>` содержит `int8_t`, `uint8_t`, `int16_t`, `uint16_t`, `int32_t`, `uint32_t`, `int64_t`, `uint64_t`.
- Они существуют только тогда, когда реализация действительно может предоставить целый тип ровно такой ширины.

### Специальные служебные типы

- `size_t` — размер объектов и массивов.
- `ptrdiff_t` — разность двух указателей на элементы одного массива.

### `_BitInt(N) из C23`

- Семейство целочисленных типов произвольной битности.
- Есть знаковая и беззнаковая формы: `_BitInt(N)` и `unsigned _BitInt(N)`.
- Поддержка зависит от компилятора.

## Типы с плавающей точкой

Плавающая точка в C тесно связана с IEEE-754 и архитектурой платформы.

### `float`

- Размер `float` в байтах стандартом не фиксирован.
- По возможностям представления тип `float` не "хуже", чем формат с минимум 6 десятичными значащими цифрами точности.
- Гарантии обычно описывают через `<float.h>`, например `FLT_DIG >= 6`.
- Обычно соответствует single precision.
- Подходит для экономии памяти и пропускной способности, но имеет ограниченную точность.

### `double`

- Размер `double` в байтах стандартом не фиксирован.
- По точности и диапазону `double` не уже, чем `float`.
- Минимальная гарантия по точности: примерно 10 десятичных значащих цифр, то есть `DBL_DIG >= 10`.
- Стандартный рабочий тип для многих вычислений.
- Обычно соответствует double precision.

### `long double`

- Размер `long double` в байтах стандартом не фиксирован.
- По точности и диапазону `long double` не уже, чем `double`.
- Минимальная гарантия по точности не хуже `double`: `LDBL_DIG >= 10`.
- Может быть реализован как тот же `double`, 80-битный extended precision или более широкий формат.

### `_Float16`

- Появился в C23.
- Соответствует half precision там, где поддерживается реализацией.
- Полезен там, где критичны плотность хранения и пропускная способность, но нужно помнить о сильном падении точности.

### Half precision и родственные форматы

Во второй половине курса всплывают и другие форматы:

- half precision;
- bfloat16;
- сверхкороткие форматы вроде `e5m2` и `e4m3`.

Они важны для архитектуры вычислений, ускорителей и ИИ, но уже требуют особенно осторожного отношения к точности, переполнению и поведению специальных значений.

### Комплексные числа

- В C есть поддержка комплексных чисел, например `_Complex float`.
- Это не "магия языка", а полноценный числовой тип со своей арифметикой и библиотечной поддержкой.

## Соотношения размеров, которые гарантирует стандарт

Для стандартных целочисленных типов полезно помнить общую цепочку:

`sizeof(char) <= sizeof(short) <= sizeof(int) <= sizeof(long) <= sizeof(long long)`

Так как `sizeof(char) == 1`, все остальные размеры выражаются в байтах относительно него.

При этом стандарт не требует, чтобы байт был ровно 8 бит, хотя на практике это почти всегда так.

## Спецификаторы форматирования

Спецификатор форматирования должен соответствовать типу значения. Несовпадение формата и фактического типа — типичная причина ошибок.

### `%i`

- Используется для `int`.
- В `scanf` читает целое число и пропускает ведущие пробелы.

### `%u`

- Используется для `unsigned int`.

### `%lld`

- Используется для `long long int` в десятичной записи.

### `%lli`

- Также относится к `long long int`.
- Часто используется при вводе через `scanf`.

### `%llx`

- Используется для шестнадцатеричного представления `long long int`.

```c
long long int x;
unsigned int y;

scanf("%lld", &x);
scanf("%lli", &x);
scanf("%llx", &x);
scanf("%u", &y);
```

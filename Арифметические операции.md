## Обычные операции

К базовым арифметическим операциям относятся:

- `+` — сложение;
- `-` — вычитание;
- `*` — умножение;
- `/` — деление;
- `%` — остаток от деления.

Даже у самых базовых операций есть свои особенности:
- если в выражении участвуют целый и вещественный тип, результат приводится к вещественному;
- если вместе участвуют `signed` и `unsigned`, выражение может быть приведено к беззнаковому типу;
- если оба аргумента целочисленные, то результат тоже целочисленный. Поэтому `int / int` дает **целочисленное деление**;
- остаток от деления не математический, то есть остаток от деления на `n` может быть   `-n+1, -n+2, ..., n-2, n-1`.
Примеры:
```c
5 / 2   == 2
-5 / 2  == -2
-5 % 2  == -1
```

В C деление целых округляет результат **к нулю**, а не вниз (см. [[АрхЭВМ#Методы округления|округление]]).

Как получить UB:
- деление на ноль;
- выражение вида `MIN_INT / (-1)`.

## Побитовые операции

Побитовые операции работают с отдельными битами целых чисел.

- `&` — побитовое AND;
- `|` — побитовое OR;
- `^` — побитовое XOR;
- `~` — побитовое NOT;
- `<<` — сдвиг влево;
- `>>` — сдвиг вправо.

Что полезно помнить:
- эти операции применяются к **целочисленным типам**;
- `<<` ведёт себя как умножение на степень двойки;
- `>>` ведёт себя как деление на степень двойки.

Особенность сдвигов:
- побитовые сдвиги — это единственные бинарные операции, для которых аргументы не обязаны иметь один и тот же тип в привычном смысле, например можно сдвигать `long` на `int`.
- существует два вида сдвигов вправо: логический и арифметический;
- логический сдвиг вправо работает для беззнаковых типов, удаляя младший бит и ставя 0 в старший бит;
- арифметический сдвиг вправо работает для знаковых типов, удаляя младший бит и копируя в старший бит ПРЕДЫДУЩИЙ старший бит.

Примеры:
```c
uint8_t a = 0b10000011; // 10000011 = 131
a >>= 1;                // 01000001 = 65
int8_t b = 0b10000011;  // 10000011 = -125
b >>= 1;                // 11000001 = -63
```

Как получить UB:
 - Сдвиг чисел больше, чем на их длину.
 - Сдвиг на отрицательное число.

## Логические операции

Логические операции работают с истинностью выражений.

- `&&` — логическое И;
- `||` — логическое ИЛИ;
- `!` — логическое НЕ.

Перед вычислением аргументы логических операций приводятся к логическому смыслу:

- `0` считается ложью;
- ненулевое значение считается истиной.

Главная особенность `&&` и `||` — **неполное вычисление**:

- у `a && b` сначала вычисляется `a`, и если оно ложно, `b` уже не вычисляется;
- у `a || b` сначала вычисляется `a`, и если оно истинно, `b` уже не вычисляется.

Пример:
```c
if (x != 0 && 10 / x > 1) {
    printf("ok\n");
}
```

Здесь вторая часть условия не будет вычисляться, если `x == 0`, и это защищает код от деления на ноль.

Важно:

- это свойство характерно именно для `&&` и `||`;
- для многих других бинарных операций порядок вычисления аргументов в C не фиксирован.

## Инкременты

Существует 2 типа инкрементов:
 - `i++` - постфиксный инкремент. Сначала использовать значение `i`, а потом увеличить его на `1`.
 - `++i` - префиксный инкремент. Сначала увеличить значение на `1`, а потом использовать увеличенное значение `i`.
 - Понятно, что при использовании инкремента в качестве увеличения переменной на `1` инкременты не будут иметь различия.
 - Аналогично существуют постфиксный декремент `i--` и префиксный декремент `--i`.
 
 ```C
 int a = 5, b = 5;
 c = a++; // Тогда a = 6, c = 5
 c = ++b; // Тогда b = 6, c = 6
 ```

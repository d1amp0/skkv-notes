Во второй лекции отдельно обсуждалось, как C выполняет обычные арифметические, побитовые и логические операции. Этот раздел собран как краткий справочник по ним.

--------------------------------------------------------------------------------
## Обычные операции

К базовым арифметическим операциям относятся:

- `+` — сложение;
- `-` — вычитание;
- `*` — умножение;
- `/` — деление;
- `%` — остаток от деления.

Важно помнить несколько правил:

- если в выражении участвуют целый и вещественный тип, результат приводится к вещественному;
- если вместе участвуют `signed` и `unsigned`, выражение может быть приведено к беззнаковому типу;
- если оба аргумента целочисленные, то `int / int` дает **целочисленное деление**.

Примеры:

```c
5 / 2   == 2
-5 / 2  == -2
-5 % 2  == -1
```

В C деление целых округляет результат **к нулю**, а не вниз.

Нужно быть осторожным:

- деление на ноль — это undefined behaviour;
- выражение вида `MIN_INT / (-1)` для знакового типа тоже может приводить к undefined behaviour.

## Побитовые операции

Побитовые операции работают с отдельными битами целых чисел.

- `&` — побитовое AND;
- `|` — побитовое OR;
- `^` — побитовое XOR;
- `~` — побитовое NOT;
- `<<` — сдвиг влево;
- `>>` — сдвиг вправо.

Что полезно помнить:

- эти операции применяются к **целочисленным типам**;
- `<<` часто ведет себя как умножение на степень двойки;
- `>>` для беззнаковых чисел и положительных знаковых часто похоже на деление на степень двойки;
- сдвиги для отрицательных знаковых значений требуют осторожности и могут зависеть от реализации.

Особенность сдвигов:

- побитовые сдвиги — это единственные бинарные операции, для которых аргументы не обязаны иметь один и тот же тип в привычном смысле, например можно сдвигать `long` на `int`.
--------------------------------------------------------------------------------
## Логические операции

Логические операции работают с истинностью выражений.

- `&&` — логическое И;
- `||` — логическое ИЛИ;
- `!` — логическое НЕ.

Перед вычислением аргументы логических операций приводятся к логическому смыслу:

- `0` считается ложью;
- ненулевое значение считается истиной.

Главная особенность `&&` и `||` — **неполное вычисление**:

- у `a && b` сначала вычисляется `a`, и если оно ложно, `b` уже не вычисляется;
- у `a || b` сначала вычисляется `a`, и если оно истинно, `b` уже не вычисляется.

Пример:

```c
if (x != 0 && 10 / x > 1) {
    printf("ok\n");
}
```

Здесь вторая часть условия не будет вычисляться, если `x == 0`, и это защищает код от деления на ноль.

Важно:

- это свойство характерно именно для `&&` и `||`;
- для многих других бинарных операций порядок вычисления аргументов в C не фиксирован.

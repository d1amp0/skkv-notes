## Комментарии в C

В C используются две основные формы комментариев.

### Однострочный комментарий

```c
// комментарий до конца строки
```

### Многострочный комментарий

```c
/*
   комментарий
   на несколько строк
*/
```

Важно:

- вложенные `/* ... */` комментарии не поддерживаются;
- комментарии участвуют в ранних стадиях обработки исходника, поэтому стоит осторожно обращаться с ними внутри макросов и условной компиляции.

## Склейка строк через обратный слеш

На раннем этапе трансляции последовательность `\` в конце строки склеивает текущую строку со следующей.

Пример:

```c
int x = 12345\
67890;
```

Это корректный код, и компилятор увидит здесь одно число `1234567890`.

Этот механизм работает до обычного синтаксического разбора, поэтому влияет не только на строки, но и на макросы, комментарии и другие токены.

## __PRETTY_FUNCTION__

`__PRETTY_FUNCTION__` — популярное расширение компилятора, которое дает строку с именем текущей функции, а иногда и с более подробной сигнатурой.

Пример:

```c
void f(int *p) {
    printf("%s\n", __PRETTY_FUNCTION__);
}
```

Это удобно для:

- отладки;
- логирования;
- различения перегруженных или параметризованных функций в C++-подобных режимах компиляции.

Важно:
- это не часть стандартного C;
- формат строки зависит от компилятора.

## Endian-ность

Endian-ность описывает порядок хранения байтов многобайтового значения в памяти.

### Little-endian

- Младший байт лежит по меньшему адресу.

### Big-endian

- Старший байт лежит по меньшему адресу.

Почему это важно:

- при записи бинарных файлов;
- при сетевом обмене;
- при ручной интерпретации памяти через указатели, массивы байтов и `fread`/`fwrite`.

Например, одно и то же число `0x12345678` будет лежать в памяти по-разному на little-endian и big-endian машинах.

В языке C тип определяет, как интерпретируется память, какие операции допустимы над значением и какие соглашения ввода-вывода нужно соблюдать.

## Базовые типы

### `int`

- Базовый знаковый целочисленный тип.
- Исторически стандарт требовал диапазон не хуже `-32767 ... 32767`, а современные реализации обычно дают значительно больше.
- Локальные переменные типа `int` без инициализации содержат неопределенное значение.
- Глобальные объекты без явной инициализации автоматически нулевые.

```c
int g; // глобальный объект, инициализируется нулем

int main(void) {
    int x;      // неинициализирован
    int y = 5;  // инициализирован
    return 0;
}
```

### void

- Обозначает отсутствие значения.
- Используется как тип возвращаемого значения функции без результата.
- В списке параметров `void` означает, что функция не принимает аргументов.

```c
void log_message(void) {
    printf("ok\n");
}
```

## Дополнительные целочисленные типы

### unsigned int

- Беззнаковая версия `int`.
- Переполнение для беззнаковых типов определено стандартом как арифметика по модулю.

### short int

- Тип не "длиннее", чем `int`.
- Используется реже, чем ожидают новички, и ценен в основном тогда, когда важен формат хранения.

### long int

- Тип не меньше `int`.
- Его размер зависит от платформы, поэтому `long` не лучший выбор для переносимого бинарного формата.

### long long int

- Тип не меньше `long`.
- На практике обычно дает как минимум 64 бита.

### char

- Тип, близкий к одному байту.
- У обычного `char` знаковость не фиксирована стандартом.
- Если нужен именно числовой смысл, лучше явно использовать `signed char` или `unsigned char`.

### _Bool и bool

- `_Bool` появился в C99.
- Хранит логическое значение: по смыслу `0` или `1`.
- Имена `bool`, `true`, `false` традиционно дает `<stdbool.h>`.
- Начиная с C23 работа с `bool` стала удобнее.

### Типы фиксированного размера

- `int32_t`, `uint32_t` и родственные типы объявляются в `<stdint.h>`.
- Используются там, где размер объекта должен быть задан явно.

### Специальные служебные типы

- `size_t` — размер объектов и массивов.
- `ptrdiff_t` — разность двух указателей на элементы одного массива.

### _BitInt(N) из C23

- Семейство целочисленных типов произвольной битности.
- Есть знаковая и беззнаковая формы: `_BitInt(N)` и `unsigned _BitInt(N)`.
- Поддержка зависит от компилятора.

## Типы с плавающей точкой

Плавающая точка в C тесно связана с IEEE-754 и архитектурой платформы.

### float

- Обычно соответствует single precision.
- Подходит для экономии памяти и пропускной способности, но имеет ограниченную точность.

### double

- Стандартный рабочий тип для многих вычислений.
- Обычно соответствует double precision.

### long double

- Не меньше `double`.
- Может быть реализован как тот же `double`, 80-битный extended precision или более широкий формат.

### _Float16

- Появился в C23.
- Соответствует half precision там, где поддерживается реализацией.
- Полезен там, где критичны плотность хранения и пропускная способность, но нужно помнить о сильном падении точности.

### Half precision и родственные форматы

Во второй половине курса всплывают и другие форматы:

- half precision;
- bfloat16;
- сверхкороткие форматы вроде `e5m2` и `e4m3`.

Они важны для архитектуры вычислений, ускорителей и ИИ, но уже требуют особенно осторожного отношения к точности, переполнению и поведению специальных значений.

### Комплексные числа

- В C есть поддержка комплексных чисел, например `_Complex float`.
- Это не "магия языка", а полноценный числовой тип со своей арифметикой и библиотечной поддержкой.

## Спецификаторы форматирования

Спецификатор форматирования должен соответствовать типу значения. Несовпадение формата и фактического типа — типичная причина ошибок.

### %i

- Используется для `int`.
- В `scanf` читает целое число и пропускает ведущие пробелы.

### %u

- Используется для `unsigned int`.

### %lld

- Используется для `long long int` в десятичной записи.

### %lli

- Также относится к `long long int`.
- Часто используется при вводе через `scanf`.

### %llx

- Используется для шестнадцатеричного представления `long long int`.

```c
long long int x;
unsigned int y;

scanf("%lld", &x);
scanf("%lli", &x);
scanf("%llx", &x);
scanf("%u", &y);
```

- `\\` — вывод символа обратного слеша.

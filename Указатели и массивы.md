Это один из ключевых разделов языка C. Именно здесь язык перестает быть "просто синтаксисом" и начинает напрямую работать с моделью памяти.

## Что такое указатель

Указатель — это объект, который хранит адрес другого объекта или функции.

```c
int x = 10;
int *p = &x;
```

Здесь:

- `&x` — взятие адреса;
- `p` — указатель на `int`;
- `*p` — разыменование, то есть доступ к объекту по адресу.

Важно не путать:

- `int *p` — указатель на `int`;
- `*p = 5` — запись в объект, на который указывает `p`.
--------------------------------------------------------------------------------
## Массивы

Массив — это непрерывная последовательность элементов одного типа.

```c
int arr[10];
```

Полезные свойства:

- индексация `arr[i]` эквивалентна `*(arr + i)`;
- в большинстве выражений массив преобразуется к указателю на первый элемент;
- сам массив и указатель — не одно и то же.

Пример:

```c
int arr[10];
int *p = arr;
```

Здесь `p` указывает на `arr[0]`, но `sizeof(arr)` и `sizeof(p)` — разные вещи.

--------------------------------------------------------------------------------
### Инициализация массивов

```c
int a[10];              // неинициализированные элементы
int b[10] = {};         // все нули
int c[5] = {1, 2};      // остальные элементы станут нулями
int d[10] = {[5] = 5};  // designated initializer
```
--------------------------------------------------------------------------------
### Многомерные массивы

```c
int z[5][3];
```

Это не "массив указателей", а единый блок памяти с фиксированным внутренним шагом.

При передаче двумерного массива в функцию нужно знать размер всех измерений, кроме первого:

```c
void f(int a[][3]);
void g(int (*a)[3]);
```
--------------------------------------------------------------------------------
## Динамическая память

Когда размер данных неизвестен на этапе компиляции или объект должен жить дольше текущего блока, используют динамическую память.

--------------------------------------------------------------------------------
### `malloc`

```c
int *p = malloc(5 * sizeof(int));
```

- выделяет блок памяти;
- возвращает `void *`, который в C не нужно явно приводить;
- при неудаче возвращает `NULL`.
--------------------------------------------------------------------------------
### `calloc`

```c
int *p = calloc(5, sizeof(int));
```

- выделяет память под несколько элементов;
- дополнительно заполняет ее нулями.
--------------------------------------------------------------------------------
### `realloc`

```c
int *tmp = realloc(p, 20 * sizeof(int));
if (tmp) {
    p = tmp;
}
```

- пытается изменить размер ранее выделенного блока;
- может вернуть новый адрес;
- при ошибке старый блок остается валиден, если возврат был `NULL`.
--------------------------------------------------------------------------------
### `free`

```c
free(p);
p = NULL;
```

Важно:

- `free` не зануляет сам указатель;
- повторный `free(p)` после освобождения — ошибка;
- `free(NULL)` допустим и безопасен.
--------------------------------------------------------------------------------
## Типичные ошибки

- разыменование `NULL`;
- use-after-free;
- double free;
- потеря указателя на выделенную память;
- выход за границы массива;
- путаница между размером массива и размером указателя.
--------------------------------------------------------------------------------
## VLA и указатели на массив

В C встречаются массивы переменной длины и указатели на массивы фиксированной ширины.

```c
int M = 10, K = 20;
int (*ptr)[M] = malloc(M * K * sizeof(int));
```

Такие формы особенно важны для двумерных данных, где нужно сохранить правильную арифметику индексов.

## Строки в C

Строка в C — это массив `char`, заканчивающийся нулевым байтом `'\0'`.

```c
"abc"
```

На самом деле это последовательность:

```text
'a' 'b' 'c' '\0'
```

Из этого следуют важные свойства:

- `sizeof("abc") == 4`;
- `strlen("abc") == 3`;
- длина строки определяется не из метаданных, а поиском терминатора.

Подход с нуль-терминатором:

- не требует отдельного поля длины;
- делает вычисление длины операцией `O(n)`;
- требует аккуратной работы с буферами.
--------------------------------------------------------------------------------
## Передача массивов в функции

Когда массив передают в функцию, параметр фактически становится указателем.

Эти объявления по смыслу эквивалентны:

```c
void f1(int a[10]);
void f2(int a[]);
void f3(int *a);
```

Именно поэтому внутри функции уже нельзя узнать исходную длину массива через `sizeof(a)`.

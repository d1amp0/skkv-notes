## if

`if` выполняет ветвление по условию.

```c
if (x > 0) {
    printf("positive\n");
} else if (x < 0) {
    printf("negative\n");
} else {
    printf("zero\n");
}
```

Условие интерпретируется по правилам C:

- `0` — ложь;
- любое ненулевое значение — истина.

## Циклы: while, do while, for

### `while`

Выполняет тело цикла, пока оно верно. При этом сначала проверяется условие, а потом переходим в тело цикла.

```c
while (x > 0) {
    --x;
}
```

### `do while`

Аналогичо обычному циклу `while`, но сначала выполняется тело цикла, а потом проверяется условие.
```c
do {
    read_next();
} while (need_more());
```

### `for`

Компактная запись цикла с инициализацией, условием и шагом. Необязательно писать заполнять все 3 части цикла:
```c
for (int i = 0; i < n; ++i) {
    printf("%i\n", i);
}

int i = 0;
for (; i < n; i++) {
	// ...
}

for (int i = 0; i < n; ) {
	// ...
	i += 2;
}

for (;;) {
	// Бесконечный цикл
}
```

## `break`, `continue`

Для циклов есть полезные ключевые слова:

- `break` завершает БЛИЖАЙШИЙ цикл или `switch`;
- `continue` переходит к следующей итерации цикла;
--------------------------------------------------------------------------------
## switch

`switch` удобен, когда нужно выбрать одно из нескольких действий по целочисленному или перечислимому значению.

```c
switch (x) {
case 1:
    printf("one\n");
    break;
case 2:
    printf("two\n");
    break;
default:
    printf("other\n");
}
```

Что важно помнить:

- без `break` выполнение "проваливается" в следующий `case`;
- `default` необязателен, но полезен для полноты логики;
- `switch` часто работает вместе с `enum`.

Также стоит упомянуть `Duff's device` — необычный прием, использующий проваливание `case` внутри цикла для ручного разворачивания. Тем не менее, эта оптимизация в наше время больше не используется, потому что компиляторы и так уже стали работать эффективнее.

## goto и метки

`goto` делает безусловный переход к метке внутри той же функции.

```c
if (error) {
    goto cleanup;
}

cleanup:
free(ptr);
```

Хотя `goto` часто считают "грязной" конструкцией, в системном C он бывает удобен для единообразной очистки ресурсов при ошибках (как в примере выше).
При этом с помощью `goto` нельзя пропускать инициализацию переменных.
